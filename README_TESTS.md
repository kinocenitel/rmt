# Автотесты Risk Matrix

Автотесты на Python + Playwright. Запускают инструмент в браузере
(Chromium/Firefox/WebKit) через локальный http-сервер и проверяют его.

## Что нужно установить

Один раз на компьютере:

### 1. Python 3.10+

Если у тебя macOS, он, скорее всего, уже есть. Проверить:

```bash
python3 --version
```

Если нет — [скачай с python.org](https://www.python.org/downloads/) или через
Homebrew: `brew install python@3.12`.

### 2. Зависимости проекта

Из корня проекта (где лежит `index.html`):

```bash
python3 -m venv .venv          # создаём изолированное окружение
source .venv/bin/activate      # активируем его (на Windows: .venv\Scripts\activate)
pip install -r requirements.txt
```

### 3. Браузеры для Playwright

Playwright сам скачивает нужные версии браузеров — это отдельная команда:

```bash
playwright install chromium
```

(Первый раз качает ~150 МБ. Если нужны все браузеры — `playwright install`.)

## Как запускать тесты

Всегда из корня проекта и с активированным `.venv`.

### Все тесты

```bash
pytest
```

### Только smoke-набор

```bash
pytest -m smoke
```

### Один конкретный тест

```bash
pytest tests/test_smoke.py::test_add_single_risk
```

### В видимом браузере (полезно для отладки)

По умолчанию тесты идут в headless-режиме (без окна браузера). Чтобы увидеть,
что происходит:

```bash
pytest --headed
```

### С замедлением действий (slowmo), чтобы успеть разглядеть

```bash
pytest --headed --slowmo 900
```

## Что делают фикстуры

Каждый тест получает свежий browser-контекст (изолированный localStorage),
и локальный http-сервер уже запущен на фоне.

Есть две основные стартовые ситуации:

- **`rm`** — страница с пустым списком рисков (0 активных, 0 архив).
  Это используется в 99% тестов.
- **`rm_fresh`** — страница в состоянии «первый запуск», с 8 демо-рисками.
  Используется только в одном тесте: проверка дефолтного поведения.

Фикстуры `xlsx_builder.valid() / .empty() / .oversized()` создаются во
временной папке xlsx-файлы для импортных тестов.

## Структура

```
tests/
├── conftest.py              ← фикстуры (http-сервер, браузер, xlsx)
├── pages/
│   └── risk_matrix_page.py  ← Page Object (селекторы и действия)
└── test_smoke.py            ← первые 10 smoke-тестов
```

## Частые проблемы

**`playwright: command not found`** — ты не активировал `.venv` или забыл
`pip install -r requirements.txt`.

**Браузер не открывается в `--headed` режиме** — на macOS может попросить
разрешение в «Настройках безопасности» при первом запуске.

**Тесты падают с `Timeout 30000ms exceeded`** — обычно проблема с селектором
или с фикстурой состояния. Посмотри на скриншот (Playwright их делает при
падении) в папке `test-results/`.

## CI: автоматический прогон на GitHub

В репозитории есть `.github/workflows/tests.yml` — он автоматически запускает
тесты при:
- push в ветку `main`;
- открытии pull request в `main`;
- ручном запуске через вкладку Actions в репозитории.

Прогон идёт на Ubuntu, в том же окружении, что и локально. Chromium
кэшируется между прогонами, поэтому после первого прогона повторные
идут ~40 секунд вместо ~2 минут.

При падении теста в артефактах прогона сохраняются скриншоты и видео —
их можно скачать из интерфейса GitHub и посмотреть глазами, что случилось: вкладка Actions → конкретный прогон → раздел Artifacts.
